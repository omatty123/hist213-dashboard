## WHAT "ENHANCED" MEANS

An enhanced plate JSON includes ALL of these fields:

### Required Fields
1. **readingAnalyses** - Array of per-reading breakdowns:
   - title, author, source, pages, type
   - summary (2-3 paragraphs)
   - keyQuotes (3-4 quotes with page numbers)
   - thematicExamples (3 concrete examples from text)
   - realLifeConnections (3 modern parallels)

2. **overallSynthesis** - How the readings work together (2-3 paragraphs)

3. **executiveSummary** - What students should take away (1-2 paragraphs)

4. **keywords** - Array with term, definition, relevance

5. **keyPassages** - Best quotes with source and page

6. **peopleplacesthings** - People, places, things arrays

7. **keyQuestions** - Discussion questions with talking points

8. **timeline** - Date/event pairs

9. **connections** - fromPrevious, toNext, broaderThemes

10. **teachingNotes** - Array of practical teaching tips

11. **map** - If geographic content exists: center, zoom, locations array

## HOW TO RESUME THIS PROJECT

When starting a new Claude Code session:

1. Read this file: `~/.claude/teaching/dashboard/prep/PROJECT_STATE.md`
2. Check which plates are PENDING
3. For each PENDING plate:
   - Read the source PDF from the appropriate Part_X file
   - Read the existing plate JSON
   - Generate enhanced content following the template in plate-5.json
   - Write the updated JSON
   - Update this file to mark plate as DONE

### Commands
```
# Read project state
cat ~/.claude/teaching/dashboard/prep/PROJECT_STATE.md

# Check plate structure
cat ~/.claude/teaching/dashboard/prep/plate-5.json | head -50

# Run /prep skill for automated generation
/prep plate 6
```
